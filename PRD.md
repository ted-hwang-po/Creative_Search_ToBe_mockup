# Creative Search ToBe — Product Requirements Document (PRD)

> 이 문서는 기존 프로토타입 코드베이스를 역분석하여 생성한 PRD입니다.

---

## 1. 제품 개요

### 1.1 제품명
**Creative Search** — 광고 소재 검색 결과 요약 및 인사이트 플랫폼 (프로토타입)

### 1.2 제품 비전
광고 소재 검색 결과를 단순 나열이 아닌, **요약·인사이트·실행 제안(A/B 테스트, 소재 기획)**까지 한 화면에서 이해하고 다음 행동으로 이어갈 수 있는 통합 소재 분석 도구

### 1.3 대상 사용자
- 뷰티/커머스 브랜드의 **퍼포먼스 마케터** 및 **소재 기획자**
- 광고 크리에이티브의 트렌드를 파악하고 다음 소재를 기획해야 하는 실무자
- 자사 브랜드와 경쟁 브랜드의 소재 전략을 비교 분석해야 하는 담당자

### 1.4 핵심 가치 제안
1. **검색 결과의 구조화된 요약**: 수백 개의 소재를 브랜드별·유형별·성분별로 요약
2. **실행 가능한 인사이트**: 단순 데이터 나열이 아닌, 행동 제안형 문구와 A/B 테스트 세팅 제공
3. **다차원 분석**: 프로모션, 키 메시지, 인플루언서, 성분/제형/효과 트렌드, 월간 트렌드 등 다양한 관점의 분석

## 2. 배경 및 문제 정의

### 2.1 현재 상황
- 광고 소재 검색 시 결과가 단순 리스트로 나열되어, 전체적인 패턴이나 트렌드를 파악하기 어려움
- 브랜드별 소재 전략 비교가 수작업으로 이루어지고 있음
- A/B 테스트 설계 시 근거가 부족하고, 어떤 소재 유형을 테스트해야 할지 직관에 의존

### 2.2 사용자 피드백 (v2 반영 사항)
1. 성분 강조 결과에서 **성분별 소재 특징**이 필요
2. "클러스터 보기"를 **특징 설명 + 썸네일 그리드 모달**로 업데이트
3. A/B 테스트 제안의 **근거를 더 명확히** 제시
4. 레이블 조합 나열 대신 **추천 중심 + 상세는 액션 시 노출**
5. A/B안은 실루엣이 아니라 **동일 상품/프로모션 기준의 실제 타입 예시(여러 애셋)**
6. "클러스터" 용어가 어려움 → **소재 유형/타입/그룹** 등 쉬운 용어로 변경
7. **자사 vs 경쟁**을 한눈에 비교하는 화면 구성 필요

## 3. 목표 (Goals)

### 3.1 프로토타입 목표
- 검색 결과 요약 UI를 빠르게 검증
- 5개 화면(페이지)을 통해 다양한 분석 관점을 탐색
- 사용자 피드백을 반영한 v2 데이터셋 및 UI 패턴 적용

### 3.2 UX 목표
- 기본 화면은 **추천 위주로 간결하게**, 근거/디테일은 **사용자 액션(클릭/펼치기)**으로 제공
- A/B 제안은 "왜 A/B인가"와 "무엇을 만들면 되는가(타입/애셋 예시)"가 즉시 이해되게 구성
- 자사/경쟁 비교는 동일 축(같은 스케일)로 한 화면에서 비교 가능

### 3.3 Non-Goals (프로토타입 제외 범위)
- 실제 검색 인덱싱/랭킹(semantic/lexical/hybrid) 구현
- 실제 성과 로그/광고 계정 데이터 연결
- 실제 크리에이티브 이미지/비디오 (SVG 썸네일로 대체)
- AI 채팅의 동적 응답 (고정 문구만 반환)

## 4. 페이지별 기능 요구사항

### 4.1 Page 1: 기존 페이지 — 프로모션 소재 검색 결과 요약
**경로**: `#creative-search`
**파일**: `src/pages/creative-search.jsx`

#### 4.1.1 개요
"지난 올리브영 세일 기간 소재" 같은 프로모션 컨텍스트의 검색 결과를 요약·추천·비교하는 메인 페이지

#### 4.1.2 정보 구조 (IA)

| 영역 | 구성 요소 |
|------|-----------|
| **Top Bar** | 페이지 타이틀/설명, 탭(요약/프로모션/파트너십), 검색어/총 건수/검색 시간 pill |
| **요약 탭** | 검색결과 인사이트 요약(접힘 기본), 추천 소재 유형 Top 2 + A/B 테스트 세팅, 브랜드별 요약, 검색 결과 리스트 |
| **프로모션 탭** | 기간 평균 성과 KPI(매출/공헌이익/CTR/CVR/ROAS), 우수 소재 Top 5, 경쟁 브랜드 카드 |
| **파트너십 탭** | 브랜드별 파트너십 소재 요약(인플루언서/공통 카피/레이아웃/예시 소재) |
| **플로팅** | 최상단 버튼, AI Chat 패널(고정 응답) |

#### 4.1.3 핵심 기능
- **검색결과 인사이트 요약**: 기본 접힘(fold) 상태, 펼치면 카드별 1문장 인사이트 + 상세
- **추천 소재 유형 Top 2**: 점수 기반 상위 2개 유형 + A/B 테스트 세팅(예산/기간/목적/지면/그룹당 소재)
- **A/B 근거 블록**: 기본 접힘, 펼치면 추천 이유 Top 3 + 지원 데이터 + 대표 예시
- **A/B안 실제 타입 예시**: 그룹별 타입 2개, 타입당 애셋 예시 2~3개 (필수 요소 + 변형 가이드)
- **브랜드별 요약**: 미디어/광고유형 비중 차트, 반복 특징 Top 2, 소재 유형 리스트
- **자사 vs 경쟁 비교**: 축 선택 토글(영상 비중/파트너십 비중/Top 레이아웃 등), 동일 스케일 비교 UI
- **모달**: 소재 상세 모달, 소재 유형 모달(좌: 설명+근거 / 우: 썸네일 그리드 6~12개)

#### 4.1.4 데이터 요구사항
- 브랜드 5개(자사 1 + 경쟁 4): 올리브영, 이니스프리, 라비오뜨, 라운드랩, 헤라
- 프로모션: "올영 세일" (2026-02-28 ~ 2026-03-06)
- 결과 수: 총 300+ (UI pill 표시), 리스트는 상위 일부만 노출
- 인플루언서 4명 (파트너십 탭용)
- 레이아웃 라이브러리: SVG 실루엣 5종

### 4.2 Page 2: 키 메시지(성분/구성) 관심사
**경로**: `#creative-search-key-message`
**파일**: `src/pages/creative-search-key-message.jsx`

#### 4.2.1 개요
"성분이나 제품 구성을 강조한 소재들은 보통 어떻게 생겼어?" 같은 질문형 검색에 대해, 성분 강조 vs 제품 구성 강조 두 축으로 분석

#### 4.2.2 정보 구조 (IA)

| 영역 | 구성 요소 |
|------|-----------|
| **Top Bar** | 타이틀 "소재 검색 결과 (키 메시지 관심사)", pill(검색어/총건수/검색시간) |
| **인사이트** | 2줄 자연어 요약(성분 강조/제품 구성 강조 각각 1문장) |
| **검색 결과 요약** | 접힘: 수량/이미지·영상 비중 chips / 펼침: 카테고리별 레이아웃 클래스 비중 차트 |
| **브랜드별 비중** | 접힘/펼침: 각 카테고리별 브랜드 분포 차트 |
| **카드 리스트** | 상위 24개 카드 + "고효율 예상" 뱃지(runDays >= 7) |

#### 4.2.3 핵심 기능
- **두 축 비교 분석**: 결과를 `ingredient`(성분 강조) vs `composition`(제품 구성 강조)으로 분리
- **분포 시각화**: 레이아웃 클래스 비중, 브랜드 분포를 미니 바 차트로 표현
- **성분별 인사이트 (v2)**: 칩/드롭다운으로 Top 5~8 성분 선택 → 1문장 요약, 자주 쓰는 레이아웃/카피/CTA, 시각적 특징, Do/Don't, 예시 썸네일 그리드
- **일관된 SVG 썸네일**: 브랜드/포커스/미디어/레이아웃/토큰 기반 결정적 생성 (64x64 data URL)

#### 4.2.4 데이터 요구사항
- 결과 240개 (모사 데이터, 결정적 seed 기반)
- 성분별 인사이트: 상위 8개 성분 (v2 데이터셋)
- 성분: 나이아신아마이드, 레티놀, 비타민C, 세라마이드, 판테놀, 시카, 히알루론산

### 4.3 Page 3: 인플루언서 소재
**경로**: `#creative-search-influencer`
**파일**: `src/pages/creative-search-influencer.jsx`

#### 4.3.1 개요
인플루언서 게시물의 분포/특징과 브랜드별 활용 패턴을 한 화면에서 분석

#### 4.3.2 정보 구조 (IA)

| 영역 | 구성 요소 |
|------|-----------|
| **Top Bar** | 타이틀/설명 + 검색어/총건수/검색시간 pill |
| **일반 요약** | 2줄 요약 + 지표 카드 grid(비중/평균 팔로워/최근 30일 집행 수) |
| **추천 인플루언서** | 참여 빈도/고효율 예상 참여/참여율 기반 Top 리스트 |
| **고효율 인사이트** | 공통 특징 3~5개 + 카드 리스트 + 상세 모달 (fold UI) |
| **브랜드별 활용** | 등급 분포 차트(nano/micro/mid/mega) + 브랜드 카드 (fold UI) |
| **카드 리스트** | 기본 24개, 확장 시 120개 |
| **플로팅** | Chat UI + Back-to-top |

#### 4.3.3 핵심 기능
- **인플루언서 정보**: 팔로워 수, 게시물당 평균 좋아요/댓글, 스토리/릴스/피드 비중, 프로필 사진/바로가기
- **등급 분류**: nano/micro/mid/mega 팔로워 구간별 분포
- **고효율 예상**: `runDays >= 7` 조건 통일
- **Chat 흐름**: 열림 시 "적합한 인플루언서를 탐색해보시겠습니까?" → "네" 입력 시 "정식 제품에서는 인플루언서 시딩도 도와드릴게요."

#### 4.3.4 데이터 요구사항
- 소재 420개 (인플루언서 소재 비율 약 40%)
- 인플루언서 60명
- 브랜드 4개 이상
- 운영기간: 최근 60일 내 (최근 30일 집행 수 계산용)

### 4.4 Page 4: 성분/제형/효과 트렌드 (브랜드 분석)
**경로**: `#creative-search-emphasis-trend`
**파일**: `src/pages/creative-search-emphasis-trend.jsx`

#### 4.4.1 개요
"성분/제형/효과 강조 트렌드 알려줘" 질의에 대해 브랜드별 강조 포인트와 키 메시지 타입을 분석

#### 4.4.2 정보 구조 (IA)

| 영역 | 구성 요소 |
|------|-----------|
| **Top Bar** | 타이틀/pill (검색어/현재 분석 축/총 건수/검색 시간) |
| **인사이트** | 자연어 요약 + 브랜드별 특징(Top 3~4) |
| **분포 요약** | 어떤 성분/제형/효과/메시지가 많이 강조되었나 (fold) |
| **고효율 소재** | 브랜드별 Top N (fold) |
| **공통/차별 특징** | 공통 Top + 브랜드별 Top 조합 |
| **소재 기획 제안** | 클러스터 기반 제안 카드 (클릭 시 모달) |
| **브랜드 분석** | 브랜드 카드 + pill + fold |
| **카드 리스트** | 더 보기/전체 보기 |
| **플로팅** | AI Chat + Back-to-top |

#### 4.4.3 핵심 기능
- **3축 전환**: 성분(ingredient) / 제형(formula) / 효과(effect) — Chat UI를 통한 동적 전환
- **10+ 브랜드 대규모 분석**: 12개 이상 브랜드, 1200개 소재 규모
- **카테고리 분류**: skincare / color / device
- **클러스터 정의**: `dimensionKey:token:keyMessageType:layoutClass` 조합
- **Chat 전환**: "제형" 입력 시 `dimension=formula`, "효과" 입력 시 `dimension=effect`로 페이지 내용 재계산
- **클러스터 모달 고정 문구**: "해당 메시지와 레이아웃을 기준으로 소재를 만들어볼까요?"

#### 4.4.4 데이터 요구사항
- 소재 1200개 (모사 데이터)
- 브랜드 12개 이상
- 토큰 풀: `dimension x category` 분리

### 4.5 Page 5: 월간 트렌드 (주차/시즌)
**경로**: `#creative-search-monthly-trend`
**파일**: `src/pages/creative-search-monthly-trend.jsx`

#### 4.5.1 개요
"이번 달 많이 사용된 소재 뭐야?" 질의에 대해 2026년 1월 기준 주차별/시즌별 클러스터 단위 요약

#### 4.5.2 정보 구조 (IA)

| 영역 | 구성 요소 |
|------|-----------|
| **Top Bar** | 타이틀 "소재 검색 결과 (월간 트렌드)", pill |
| **인사이트** | 2줄 자연어 요약(월간 전체 특징 + 최신 주차 트렌드) |
| **검색 결과 요약** | 접힘: 시즌 브래킷 + 주차 범위 pill / 펼침: 시즌 카드 + 주차별 카드(W1~Wn) |
| **카드 리스트** | 상위 36개 카드 |
| **오버레이** | 클러스터 모달, 소재 상세 모달 |
| **플로팅** | AI Chat(밈 추천) + Back-to-top |

#### 4.5.3 핵심 기능
- **주차 계산**: 일요일~토요일 경계, month 밖으로 확장 가능
- **시즌 브래킷**: "신년맞이 프로모션" (1/1~1/4) 고정
- **주차별 클러스터**: 주차당 3~5개 클러스터, 클러스터당 5~30개 소재
- **밈 추천 흐름**: 최신 주차에 "밈 사용" 클러스터 최소 1개 포함
  - Chat에서 "Yes" 선택 → 요약 펼침 → 밈 클러스터로 자동 스크롤 + 하이라이트(ring 스타일)
- **밈 라이브러리**: 사전 정의된 `MEME_LIBRARY` 사용

#### 4.5.4 데이터 요구사항

- 이미지:영상 비중 약 4:6

## 6. 기술 아키텍처

### 6.1 기술 스택
| 기술 | 용도 |
|------|------|
| React 18 (UMD/CDN) | UI 프레임워크 |
| Tailwind CSS (CDN) | 스타일링 |
| Babel Standalone | JSX 트랜스파일 (빌드 없음) |
| SVG (data URL) | 썸네일/실루엣 에셋 |

### 6.2 아키텍처 패턴
- **No-build 프로토타입**: 빌드 과정 없이 정적 서버에서 직접 실행
- **모듈 등록 패턴**: 각 페이지/공유 모듈이 IIFE로 `window.__APP.pages.*` / `window.__APP.ui.*`에 등록
- **Boot 로더**: `boot.js`가 순서대로 JSX 파일을 로드 → Babel 트랜스파일 → eval 실행
- **Hash 라우팅**: `window.location.hash` 기반 페이지 전환
- **결정적 모사 데이터**: seed 기반 pseudo-random으로 리로드해도 동일한 결과 보장

## 8. 썸네일(SVG) 생성 규격

### 8.1 원칙
- 동일 입력 → 동일 결과 (결정적 seed 기반)
- 타입별 시각적 차이가 즉시 인지 가능하도록 블록 구성 강제
- 실제 이미지 없이도 "여러 타입의 다른 소재"임을 인지 가능

### 8.2 타입별 시각적 구성

| 타입 (typeKey) | 시각적 특징 |
|----------------|-------------|
| **BenefitGrid** (혜택 그리드형) | 3분할 카드 블록 + 하단 CTA 칩 |
| **IngredientProof** (성분 근거형) | 제품(좌) + 근거(우) 분리, 성분 칩 |
| **BeforeAfterProof** (전/후 증거형) | 2분할 + 중앙 분리선, BEFORE/AFTER 라벨 |
| **UGCReview** (UGC 리뷰형) | 큰 텍스트 박스 + 2줄 자막 박스, 리뷰 칩 |
| **Hook3s** (3초 훅형) | 중앙 '3s' 타이포 + 재생 버튼 (영상) |

### 8.3 규격
- 해상도: v1 64x64, v2 96x96 `viewBox`
- 텍스트 제한: 브랜드 마크(2자), 타입 라벨(6자), 프로모션 키(6자), 제품명/키메시지(7~8자)
- 팔레트: 타입 블록 구조 가독성 우선 (브랜드 컬러보다 고대비)

## 9. 비기능 요구사항

### 9.1 실행 환경
- 정적 서버에서 실행 (예: `python3 -m http.server 5173 --bind 127.0.0.1`)
- 빌드 과정 불필요
- 모든 의존성 CDN 로드

### 9.2 브라우저 호환
- 밝은 UI 톤 강제 (`color-scheme: light`) — 다크모드 영향 방지
- Tailwind `line-clamp-2` 폴백 CSS 포함

### 9.3 접근성
- 모달: `role="dialog"`, `aria-modal="true"`
- Back-to-top: `aria-label="최상단으로 이동"`
- 페이지 전환: `aria-expanded`, `aria-controls`, `aria-label`
- 이미지: `alt` 텍스트, `loading="lazy"`

### 9.4 재현 가능성
- 모사 데이터: `hashStringToInt()` + `seededRand01()`로 결정적 pseudo-random
- 리로드해도 동일한 결과 보장

## 10. 성공 기준 (프로토타입 관점)

1. 사용자가 "왜 이 유형/A/B가 추천되는지"를 **근거 펼침 한 번으로 이해**할 수 있다
2. "무엇을 만들면 되는지"가 **타입/애셋 예시로 구체화**되어 보인다
3. 자사 vs 경쟁 차이를 **축 1개 선택으로 즉시 비교**할 수 있다
4. 데이터가 빈약해 보이지 않도록 **풍부한 썸네일/SVG 시각 요소**와 **다수의 브랜드/소재**가 항상 보인다
5. 5개 페이지 간 **일관된 UI 패턴**(fold/expand, pill, 차트, Chat, Back-to-top)이 유지된다
