# Creative Search (성분/제형/효과 트렌드: 브랜드 분석) 페이지 요구사항/설계 (`src/pages/creative-search-emphasis-trend.jsx`)

## 개요

이 문서는 **“성분/제형/효과 강조 트렌드” 기반**의 소재 검색 결과 요약 페이지 요구사항과 구현 설계를 정리합니다. 사용자의 질의(예: “성분/제형/효과 강조 트렌드 알려줘”)로 진입한 검색결과 페이지에서, **브랜드별 강조 포인트(성분/제형/효과)**와 **키 메시지 타입**을 분석해 **고효율 예상 소재**/공통·차별 특징/소재 기획 제안을 제공합니다.

## 목표(Why)

- 검색결과에서 “무엇(성분/제형/효과)이 강조되었는지”를 **자연어 요약 + 분포**로 설명합니다.
- 브랜드별로 **강조 토큰(성분/제형/효과) + 키 메시지 타입** 조합을 비교해 공통/차별 패턴을 파악할 수 있게 합니다.
- 실제 제품 없이도 풍부하게 보이도록 **10+ 브랜드, 1000+ 소재 규모의 데이터**로 화면을 구성합니다.

## 범위(Scope)

### 포함

- Top bar: 페이지 제목/설명 + 검색어/현재 분석 축(성분/제형/효과)/총 건수/검색 시간 pill
- “검색결과 인사이트”: 자연어 요약 + 브랜드별 특징(Top 3~4 브랜드) 요약
- “어떤 성분/제형/효과/메시지가 많이 강조되었나요?”: 분포 fold(접기/펼치기)
- “브랜드별 고효율 예상 소재 리스트”: 브랜드별 Top N(기본 5) + fold(접기/펼치기)
- “브랜드별 공통/차별 특징”: 공통 Top + 브랜드별 Top 조합 요약
- “소재 기획 제안”: 자주 등장하는 클러스터 기반 제안 카드(클릭 시 모달)
- “브랜드별 강조 포인트 분석”: 브랜드 카드 + pill + fold(브랜드별 예시 소재)
- “검색 결과 리스트”: 결과 카드 리스트 + 더 보기/전체 보기
- 모달: 클러스터 상세(예시 소재 포함) + 고정 문구
- 우측 하단: AI Chat + Back-to-top 버튼

### 제외(프로토타입 한계)

- 실제 검색/랭킹/클러스터링 모델은 포함하지 않습니다(모사 데이터/규칙 기반).
- 실제 크리에이티브 이미지/비디오는 없으며, **SVG 썸네일(data URL)**로 대체합니다.

## 핵심 요구사항(Requirements)

- **전반적인 페이지 구조 준수**: 기존 페이지 모듈 패턴(IIFE + `window.__APP.pages.*` 등록)과 UI 컴포넌트(`Pill`, `SectionHeader`, `Modal`)를 사용합니다.
- **3축 분리**: 성분/제형/효과는 서로 다른 토큰 풀을 사용하고, 스킨케어/색조/디바이스를 구분해 mock data를 생성합니다.
- **고효율 예상 표기**:
  - 텍스트는 항상 “고효율 예상”으로 통일
  - 프로토타입 규칙: `runDays ≥ 7`이면 “고효율 예상”으로 표시
- **브랜드 10개 이상**: 브랜드 리스트는 12개 이상으로 구성
- **소재 1000개 이상**: 기본 1200개를 생성하고 화면에서는 점진 노출(더 보기/전체 보기)
- **상단 자연어 요약**: 전체 Top 토큰/키 메시지 타입/레이아웃과 브랜드별 특징을 자연어로 서술
- **브랜드별 분석 카드**: 강조 토큰/키 메시지 타입을 pill 형태로 노출 + fold UI 적용
- **Chat 전환**:
  - Chat 오픈 시 “성분 외에도 제형, 효과 강조 트렌드가 궁금하신가요?” 문구 노출
  - 사용자가 `제형` 입력 시 `dimension=formula`, `효과` 입력 시 `dimension=effect`로 전환(페이지 내용 재계산)
- **클러스터 모달**: 클러스터 클릭 시 모달 고정 문구 노출
  - “해당 메시지와 레이아웃을 기준으로 소재를 만들어볼까요?”

## UI/정보 구조(IA)

- Top bar
  - 타이틀: “소재 검색 결과 (성분/제형/효과 강조 트렌드)”
  - 설명: 현재 분석 축 기준으로 요약
  - pill: 검색어 / 축 / 총 N건 / 검색 시간
- 카드 영역
  - 검색결과 인사이트(자연어 요약)
  - 분포 요약(접힘/펼침)
  - 브랜드별 고효율 예상 소재 리스트(접힘/펼침)
  - 공통/차별 특징
  - 소재 기획 제안(클러스터 기반)
  - 브랜드별 강조 포인트 분석(브랜드 카드 fold)
  - 검색 결과 리스트(더 보기/전체 보기)
- 오버레이/플로팅
  - 클러스터 모달
  - AI Chat + 최상단 버튼

## 데이터/로직 설계

### 데이터 규모

- 기본 생성 수: `buildMockResults({ total: 1200, seed })`

### 카테고리/축/토큰 풀

- 카테고리: `skincare | color | device`
- 축: `ingredient | formula | effect`
- 토큰 풀은 `dimension×category`로 분리해 “서로 다른 것”을 보장합니다.

### 주요 필드

- 소재(`creative`)
  - `id`, `brandId`, `brandName`
  - `categoryKey`(skincare/color/device)
  - `mediaType`(image/video)
  - `layoutClass`
  - `ingredientToken`, `formulaToken`, `effectToken`
  - `keyMessageType`
  - `runDays`, `predictedScore`
  - `thumbUrl`(data URL SVG)

### 고효율 예상 규칙

- `runDays ≥ 7`인 경우 “고효율 예상” pill 노출

### 클러스터 정의

- 현재 축 기준 토큰 + 키 메시지 타입 + 레이아웃 조합을 클러스터 키로 사용:
  - `clusterKey = ${dimensionKey}:${token}:${keyMessageType}:${layoutClass}`

### Chat 전환

- `dimensionKey`(ingredient/formula/effect) 상태를 변경하면:
  - 상단 인사이트 문구
  - 분포 차트
  - 브랜드 카드 Top 토큰/예시
  - 리스트 카드의 토큰 pill 및 문장
  - 클러스터/모달 내용
  가 모두 재계산됩니다.

## 썸네일(SVG) 설계

- `buildCreativeThumbDataUrl()`가 카테고리별 팔레트 + 축/키메시지/레이아웃 블록을 포함한 64×64 SVG를 생성해 `thumbUrl`로 제공합니다.
- 포함 요소:
  - 브랜드 마크(2글자)
  - 축 라벨/키 메시지 타입 chip
  - 토큰 텍스트
  - 이미지/영상별 레이아웃 블록(그리드/전후/UGC/3초훅/스펙오버레이 등)

## 주요 컴포넌트 설계

- `CreativeResultCard`: 결과 카드(토큰/키메시지/미디어/카테고리/고효율 예상/클러스터 버튼)
- `BrandInsightCard`: 브랜드 카드(토큰/키메시지 pill + fold로 예시 소재)
- `ChatPanel`: 축 전환(성분/제형/효과) + 안내 응답
- `BackToTopButton`: 스크롤 임계 이후 노출되는 최상단 이동 버튼
- `Modal`: 클러스터 모달(고정 문구 + 요약 + 예시 소재)

## 문구/표기 정책

- “고효율 예상” 용어로 통일(다른 표현 금지)
- 클러스터 모달 고정 문구:
  - “해당 메시지와 레이아웃을 기준으로 소재를 만들어볼까요?”

## 실행(로컬)

- 정적 서버로 실행합니다. 예:
  - `python3 -m http.server 5173 --bind 127.0.0.1`
- 브라우저에서 `http://127.0.0.1:5173/#creative-search-emphasis-trend`로 확인합니다.
