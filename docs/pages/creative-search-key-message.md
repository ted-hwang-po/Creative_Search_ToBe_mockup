# Creative Search (키 메시지 관심사) 페이지 요구사항/설계 (`src/pages/creative-search-key-message.jsx`)

## 개요

이 문서는 **“키 메시지(성분/제품 구성) 관심사” 기반**의 소재 검색 결과 요약 페이지 요구사항과 구현 설계를 정리합니다. 사용자의 자연어 질문을 입력으로 가정하고, 결과를 “성분 강조” vs “제품 구성 강조” 두 축으로 나눠 **패턴/분포/예시 카드**를 제공합니다.

## 목표(Why)

- 사용자의 질문(예: “성분이나 제품 구성을 강조한 소재들은 보통 어떻게 생겼어?”)에 대해 **설명 가능한 요약**을 제공합니다.
- 결과를 두 카테고리로 분해해 **레이아웃 클래스 분포**와 **브랜드 분포**를 빠르게 파악할 수 있도록 합니다.
- 정적 프로토타입에서 흔히 발생하는 “데이터가 빈약해 보임”을 방지하기 위해 **200+ 규모의 결과**와 **일관된 썸네일(자동 생성 SVG)**을 제공합니다.

## 범위(Scope)

### 포함

- 상단 Top bar: 페이지 제목/설명 + 검색어/총 건수/검색 시간 pill
- “검색결과 인사이트”: 2줄 자연어 요약(성분 강조/제품 구성 강조 각각 1문장)
- “검색 결과 요약”: fold(접기/펼치기)
  - 접힘: 성분 강조/제품 구성 강조 수량, 이미지/영상 비중 chips
  - 펼침: 각 카테고리별 레이아웃 클래스 비중(Top N) 미니 차트
- “브랜드별 비중”: fold(접기/펼치기)
  - 펼침: 각 카테고리별 브랜드 분포 미니 차트
- “검색 결과 카드 리스트”: 상위 N개 카드(기본 24개) + runDays ≥ 7 “고효율 예상” 뱃지

### 제외(프로토타입 한계)

- 실제 검색/랭킹/클러스터링 모델은 포함하지 않습니다.
- 실제 크리에이티브 이미지/비디오는 없으며, **SVG 썸네일(data URL)**로 대체합니다.

## 핵심 요구사항(Requirements)

- **질문형 검색어 노출**: `QUERY_LABEL`을 pill로 노출합니다.
- **결과량**: 충분히 풍부해 보이도록 기본 200+ 결과(현재 240)를 생성합니다.
- **두 축 요약**: 결과를 `focus = ingredient|composition`으로 분리하고, 각 축의 분포/인사이트를 제공합니다.
- **차트로 표현**: 분포는 텍스트 나열이 아닌 미니 바 차트(`MiniBarRow`)로 표현합니다.
- **일관된 썸네일**: 브랜드/포커스/미디어/레이아웃/토큰에 기반한 결정적(deterministic) SVG 썸네일을 생성합니다.
- **자연어 인사이트 문장**: 요약 상단에 성분/구성 각각 1문장씩 제공해 “이 페이지가 무엇을 말하는지”를 즉시 이해 가능하게 합니다.

## UI/정보 구조(IA)

- Top bar
  - 타이틀: “소재 검색 결과 (키 메시지 관심사)”
  - 설명: 사용자의 질문에 맞춰 성분/구성 중심 요약
  - pill: 검색어 / 총 N건 / 검색 시간
- 카드 영역
  - 검색결과 인사이트(2문장)
  - 검색 결과 요약(접힘/펼침)
  - 브랜드별 비중(접힘/펼침)
  - 검색 결과 카드 리스트(상위 24개)

## 데이터/로직 설계

### 결과 생성(모사 데이터)

- `buildMockResults({ total = 240, seed })`로 결과를 생성합니다.
- 각 결과는 아래 속성을 가집니다.
  - `brandId`, `mediaType`(image/video), `focus`(ingredient/composition), `layoutClass`, `token`
  - `runDays`(3~14), `predictedScore`(0.55~0.95), `title`, `caption`
  - `thumbUrl`: **data URL SVG** (64x64) 문자열

### 결정적 난수(재현 가능)

- `hashStringToInt()` + `seededRand01()`로 pseudo-random을 만들되, 시드 문자열로 고정하여 **리로드해도 동일한 결과**가 생성되도록 합니다.

### 분포 계산

- `distribution(items, keyFn)`이 `{ k, v, pct }` 형태로 분포를 계산하고, 상위 항목을 정렬해 반환합니다.
- 주요 분포:
  - 성분 강조/제품 구성 강조 각각의 `layoutClass` 분포(Top 8)
  - 성분 강조/제품 구성 강조 각각의 `brandId` 분포(Top 8)
  - 전체의 이미지/영상 비중

### 인사이트 문장 생성

- 성분 강조: “어떤 레이아웃 클래스가 중심인지”를 기반으로 1문장 생성
- 제품 구성 강조: 동일하게 1문장 생성
- 구현은 `topLayoutIngredient`, `topLayoutComposition`의 1등 항목을 사용합니다.

### 리스트 노출

- “검색 결과 카드 리스트”는 전체 결과에서 `predictedScore` 내림차순으로 정렬 후 상위 `listLimit = 24`개만 보여줍니다.

## 썸네일(SVG) 설계

- `buildCreativeThumbDataUrl()`이 seed 기반으로 팔레트/도형 배치를 결정해 SVG를 생성하고, 이를 `data:image/svg+xml`로 인코딩하여 `thumbUrl`로 제공합니다.
- 포함 요소:
  - 브랜드 마크(2글자)
  - 영상이면 `VID` 배지 + 플레이 아이콘
  - 토큰 텍스트(예: “레티놀”, “3종 세트”)
  - 레이아웃 클래스별 간단한 블록 레이아웃(히어로/그리드/UGC/전후/영상 훅 등)

## 주요 컴포넌트 설계

- `MiniBarRow`: 분포(%)를 컴팩트한 막대로 표시
- `CreativeResultCard`: 카드(타이틀/캡션/칩/썸네일)
- `CreativeSearchKeyMessagePage`: 페이지 루트(요약 fold 상태 관리 포함)

## 문구/표기 정책

- “고효율 예상” 뱃지는 `runDays ≥ 7` 조건으로 표시합니다(현재 프로토타입 규칙).
- 사용자의 질문은 `QUERY_LABEL`로 고정되어 있으며, 향후에는 입력 UI로 확장 가능합니다.

## 실행(로컬)

- 정적 서버로 실행합니다. 예:
  - `python3 -m http.server 5173 --bind 127.0.0.1`

## 변경/확장 포인트

- 실제 검색 결과로 교체 시:
  - `buildMockResults()`를 API 응답으로 대체
  - `focus/layoutClass/token`은 모델 추출 또는 규칙 분류로 채움
- 썸네일은 실제 크리에이티브 썸네일로 대체 가능하며, 현재 구조는 그대로 유지 가능합니다.
