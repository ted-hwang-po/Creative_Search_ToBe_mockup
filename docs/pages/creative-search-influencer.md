# Creative Search (인플루언서 소재) 페이지 요구사항/설계 (`src/pages/creative-search-influencer.jsx`)

## 개요

이 문서는 **“인플루언서 소재” 질의**로 진입하는 검색결과 요약 페이지 요구사항과 구현 설계를 정리합니다. 검색결과 내 인플루언서 게시물의 분포/특징과 브랜드별 활용 패턴을 한 화면에서 확인할 수 있도록 구성합니다.

## 핵심 요구사항(Requirements)

- **프로젝트 구조 준수**: `app.jsx` 라우팅 + 페이지 파일은 `window.__APP.pages.*` 등록 방식
- **데이터 규모**:
  - 소재(게시물) 100개 이상(프로토타입은 400+ 생성, 리스트는 기본 24 / 확장 120 노출)
  - 브랜드 4개 이상
  - 인플루언서 약 60명
- **용어 통일**: 고성과 유추 → **“고효율 예상”**만 사용
- **고효율 예상 규칙**: 모든 브랜드 공통으로 `runDays ≥ 7`이면 “고효율 예상”
- **인플루언서 정보 정의**:
  - 팔로워 수, 게시물당 평균 좋아요/댓글, 스토리/릴스/피드 비중
  - 계정 이름, 프로필 사진, 프로필 바로가기
- **인플루언서 소재 일반 요약**:
  - 화면 최상단에 요약 자연어 2줄
  - 지표 카드(비중/평균 팔로워/최근 30일 집행 수 등) 형태로 노출
- **고효율 예상 게시물 인사이트**:
  - 일반 요약 하단에 공통 특징 3~5개
  - 고효율 예상 게시물 카드 노출, 클릭 시 상세 모달
  - fold UI(접기/펼치기)
- **브랜드별 인플루언서 활용**:
  - 브랜드별 nano/micro/mid/mega 등급 분포 차트
  - 브랜드별 인플루언서 게시물 공통 특징 / 고효율 예상 공통 특징
  - 클러스터 클릭 시 모달 문구: “해당 메시지와 레이아웃을 기준으로 소재를 만들어볼까요?”
- **공통 UI**:
  - Chat UI + Back-to-top 버튼 노출(기존 페이지 패턴)
  - Chat 문구:
    - 열림 시: “적합한 인플루언서를 탐색해보시겠습니까?”
    - 사용자가 “네” 입력 시: “정식 제품에서는 인플루언서 시딩도 도와드릴게요.”

## UI/정보 구조(IA)

- Top bar: 타이틀/설명 + `검색어/총건수/검색시간` pill
- 인플루언서 소재 일반 요약: 2줄 요약 + 지표 카드 grid
- 추천 인플루언서: 참여 빈도/고효율 예상 참여/참여율 기반 Top 리스트
- 고효율 예상 인사이트(폴드): 공통 특징 3~5개 + 카드 리스트 + 상세 모달
- 브랜드별 인플루언서 활용(폴드): 등급 분포 차트 + 브랜드 카드(특징/클러스터)
- 검색 결과 카드 리스트: 기본 24개, 확장 시 120개까지 노출
- Floating: Back-to-top + Chat 패널

## 데이터/로직 설계(프로토타입)

- 결정적 모사 데이터 생성(리로드 시 동일):
  - 인플루언서 60명, 소재 420개(인플루언서 소재 비율 약 40%)
  - 운영기간(`startDate/endDate`)을 최근 60일 내로 생성하여 “최근 30일 집행 수” 계산 가능
- 주요 집계:
  - 인플루언서 소재 비중 = `인플루언서 소재 수 / 전체 소재 수`
  - 평균 팔로워/좋아요/댓글 = 인플루언서 소재 참여 인플루언서(중복 제거) 평균
  - 최근 30일 집행 수 = 운영기간이 최근 30일과 겹치는 인플루언서 소재 수
  - 브랜드별 등급 분포 = 팔로워 구간(nano/micro/mid/mega) 비중
