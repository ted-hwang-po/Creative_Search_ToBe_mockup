# Creative Search v2 썸네일/타입 생성 가이드

대상
- 데이터: `src/mock/creative-search-dataset-v2.jsx`
- 기존 참고: `src/pages/creative-search-key-message.jsx`의 `buildCreativeThumbDataUrl()`

## 목적

- “실루엣 수준”을 넘어, **동일 상품/프로모션 기준에서 ‘여러 타입의 실제 애셋’처럼 보이는 썸네일**을 생성한다.
- 타입/유형 차이가 썸네일에서 즉시 보이도록(그리드/UGC/전후/성분 근거/3초 훅 등) **구성 요소를 강제**한다.
- 프로토타입 특성상 재현 가능해야 하므로, 난수는 **결정적(seed 기반)**이어야 한다.

## 생성 원칙

### 1) Seed 설계(결정적)

- 동일 입력이면 항상 동일 썸네일이 나와야 한다.
- 권장 seed 구성:
  - `seed = "${datasetSeed}:thumb:${assetId}:${brandId}:${productId}:${promoId}:${typeKey}"`

### 2) 타입(typeKey)별 시각적 강제 요소

**BenefitGrid(혜택 그리드형)**
- 3개 카드 블록(3분할) + 하단 CTA 칩
- 숫자/바 형태 1개(혜택 강도 느낌)

**IngredientProof(성분 근거형)**
- 제품 영역(좌) + 근거 영역(우) 분리
- 성분 칩(성분명) 1개

**BeforeAfterProof(전/후 증거형)**
- 화면 2분할 + 중앙 분리선
- BEFORE/AFTER 라벨 칩 2개

**UGCReview(UGC 리뷰형)**
- 텍스트 박스(큰 캡션) + 2줄 자막 박스
- “리뷰” 칩 또는 리뷰 수치 영역

**Hook3s(3초 훅형)**
- 중앙 ‘3s’ 타이포 + 재생 버튼 느낌(영상)
- 영상 배지(VID) 표시(미디어가 video면)

## 텍스트 정책(과밀도 방지)

- 썸네일 내 텍스트는 아래 4개만 허용(최대 길이 제한 필수):
  - 브랜드 마크(2자)
  - 타입 라벨(최대 6자)
  - 프로모션 키(최대 6자)
  - 제품명/키메시지(각 최대 7~8자)
- 성분명은 IngredientProof에서만 1회 노출(최대 6자)

## 팔레트 정책(브랜드 고정 X, 타입 가독성 O)

- 브랜드 컬러를 엄격히 맞추기보다, “타입 블록 구조”가 눈에 들어오도록 **고대비 팔레트**를 사용한다.
- 팔레트는 seed 기반으로 선택하되, 동일 스코프(상품/프로모션) 내에서 과도하게 동일해 보이지 않도록 소량 지터 허용.

## 해상도/비율

- 카드 리스트/모달에서 크기가 커질 수 있으므로 64px보다 큰 **96x96** 또는 128x128 권장.
- 현재 v2 데이터셋은 96x96 `viewBox="0 0 96 96"`를 사용한다.

## 필드 매핑(데이터 → 썸네일)

- `brandName` → 브랜드 칩(2자 마크)
- `promoKey` → 프로모션 칩
- `productName` → 중앙 제품명 라벨
- `typeKey` → 타입 칩 + 블록 구성 선택
- `keyMessage` → 1줄 키메시지 라벨
- `mediaType` → video면 VID 배지 + play 아이콘
- `ctaType` → BenefitGrid에서 CTA 칩 텍스트로 사용
- `ingredientName` → IngredientProof에서만 성분 칩 텍스트로 사용

## 데이터 볼륨 가이드

- A/B 제안 화면에서 “실제로 여러 애셋을 만들었다”는 인상을 주려면:
  - 그룹당 타입 2개
  - 타입당 예시 애셋 2~3개
  - 총 8~12개 썸네일이 한 번에 보이는 것이 이상적

## 품질 체크리스트(수동)

- “근거 보기”를 펼치지 않아도 썸네일만 보고 유형이 유추 가능한가?
