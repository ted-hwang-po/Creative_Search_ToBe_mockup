# Creative Search 페이지 요구사항/설계 (`src/pages/creative-search.jsx`)

## 개요

이 문서는 **Creative Search(소재 검색 결과) 프로토타입 페이지**의 요구사항과 구현 설계를 정리합니다. 본 페이지는 “검색 결과를 요약해주는 UI”를 빠르게 검증하기 위한 정적 프로토타입이며, 실제 검색/집계/추천은 모사 데이터와 단순 규칙으로 구성됩니다.

## 목표(Why)

- 검색 결과를 단순 나열이 아니라, **요약/인사이트/실행 제안(클러스터·A/B)**까지 한 화면에서 이해 가능하도록 보여줍니다.
- 다크모드/테마 영향으로 인해 텍스트가 안 보이거나 화면이 깨지는 문제를 피하도록 **명시적으로 밝은 UI 톤**을 유지합니다.
- “프로토타입”이지만 데이터가 빈약해 보이지 않도록 **브랜드 3~4개 이상이 항상 보이는 구성**과 **풍부한 썸네일/SVG 시각 요소**를 제공합니다.

## 범위(Scope)

### 포함

- 상단 요약 탭: **요약 / 프로모션 / 파트너십** 3개 탭 구성
- “검색결과 인사이트 요약” 카드: 기본 **접힘**(fold) 상태, 펼치면 카드별 상세
- 추천 클러스터 Top 2 + A/B 테스트 세팅(예산/기간/목적/지면/그룹당 소재) 시각화
- 브랜드별 요약(소재 유형 비중, 반복 특징, 클러스터, 대표 소재 카드)
- 결과 리스트(상위 N개만 보여주되 총 결과 수는 더 큰 값으로 “보이게” 표시)
- 모달: 카드 클릭 시 상세, 모달 바깥 클릭/Esc로 닫힘(기반 UI 컴포넌트 `Modal`)
- 플로팅: “최상단” 버튼
- 우측 사이드: AI Chat 패널(고정 응답)
- ASCII 아트 대신 **SVG 이미지 파일**(레이아웃 실루엣/썸네일)

### 제외(프로토타입 한계)

- 실제 검색 인덱싱/랭킹(semantic/lexical/hybrid)은 구현하지 않고 단순 점수로 모사합니다.
- 실제 성과 로그/광고 계정 데이터 연결은 하지 않습니다.
- AI Chat은 고정 문구만 반환합니다.

## 주요 사용자 시나리오(User Stories)

- 사용자는 “지난 올리브영 세일 기간 소재” 맥락에서 결과를 열고, 상단 인사이트 요약을 빠르게 훑습니다.
- 사용자는 추천 클러스터 2개를 확인하고, A/B 테스트 세팅(기간/예산/목적/지면)을 카드 형태로 확인합니다.
- 사용자는 브랜드별 요약에서 **미디어/광고유형 비중**과 **반복 특징**을 차트로 확인합니다.
- 사용자는 특정 소재/클러스터를 클릭해 모달로 상세를 확인하고, 바깥 클릭 또는 Esc로 닫습니다.
- 사용자는 우측 AI Chat에 질문을 입력하고 Enter로 전송하며, 고정 안내 응답을 받습니다.
- 사용자는 스크롤을 내린 뒤 최상단 버튼으로 부드럽게 상단으로 이동합니다.

## UI/정보 구조(IA)

- Top Bar
  - 페이지 타이틀/설명 문구
  - 탭(요약/프로모션/파트너십)
  - 검색어/총 건수/검색 시간 pill(한 줄 유지)
- 요약 탭(Overview)
  - 검색결과 인사이트 요약(접힘 기본)
  - 추천 클러스터 + A/B 세팅
  - 브랜드별 요약
  - 검색 결과 전체 리스트(상위 일부)
- 프로모션 탭(Promo)
  - 기간 평균 성과 KPI(매출/공헌이익/CTR/CVR/ROAS)
  - 우수 소재 Top 5
  - 경쟁 브랜드 카드
- 파트너십 탭(Partnership)
  - 브랜드별 파트너십 소재 요약(인플루언서/공통 카피/레이아웃/예시 소재)

## 데이터/로직 설계

### “표시 검색어”와 “내부 쿼리” 분리

- 표시용(UX 컨텍스트): `queryLabel = "지난 올리브영 세일 기간 소재"`
- 내부 쿼리: 프로토타입에서는 특정 단어 검색이 아니라 **세일 기간에 해당하는 소재 전체**를 보여주는 것이 목표이므로 `query = ""`로 두고 기간 필터로 풀을 구성합니다.

### 결과 풀(기간 기반 필터)

- `PROMO.start ~ PROMO.end` 구간과 겹치는 소재를 `promoPool`로 구성합니다.
- 이렇게 하면 특정 검색어로 인해 결과/브랜드가 한쪽으로 쏠리는 문제를 완화하고, “기간 소재” 컨텍스트가 자연스럽게 유지됩니다.

### 총 결과 수 / 검색 시간 “보이기”

- 실제 리스트는 일부만 보여주되, 상단 pill의 “총 N건”은 `mockTotalResults`로 300+ 범위를 유지합니다.
- 검색 시간은 `mockTotalResults` 기반으로 산출해(대략 비례) 더 자연스럽게 보이도록 합니다.

### 브랜드별 요약(항상 여러 브랜드가 보이도록)

- `BRANDS`에 최소 5개 브랜드를 정의하고, `promoPool`이 여러 브랜드를 포함하도록 모사 데이터(`MOCK_CREATIVES`)를 확장합니다.
- 브랜드별 요약은 `resultsByBrand`로 그룹핑하고, 자사 브랜드(`isOwn`) 우선 정렬 후 총량 기준으로 정렬합니다.

### 차트(비중/반복 특징)

- “소재 유형 비중”: `StackedBar`, `BarRow`로 이미지/영상, 일반/파트너십 비중을 시각화합니다.
- “유의미하게 반복되는 특징”: `MiniBarRow`로 Top2를 더 컴팩트하게 보여줍니다.
- 퍼센트가 28.6%처럼 반복되어 보이는 문제를 줄이기 위해 `smoothedPct()`를 적용합니다.
  - 작은 표본에서 발생하는 분수(2/7 등)의 단조로움을 완화하기 위한 **스무딩 + 작은 지터(결정적 난수)**입니다.

### 추천 클러스터 + A/B 테스트

- 클러스터는 `MOCK_CLUSTERS_BY_BRAND`에서 가져오고, predictedClusterScore로 Top2를 선정합니다.
- `abSuggestionForCluster(cluster)`가 예산/기간/목적/지면/그룹당 소재 수를 산출합니다.
- A/B 카드에는 레이아웃 실루엣을 SVG로 보여주고, 하단 표기(문구)가 상단 정보와 불일치하지 않도록 **레이아웃 이름 기준으로 통일**합니다.

## 컴포넌트 설계(주요)

- `CreativeCard`: 소재 카드(썸네일은 `thumbUrl` SVG 우선)
- `LayoutSilhouette`: 레이아웃 실루엣(ASCII 아트 대신 `assets/silhouettes/*.svg`)
- `InsightsSummary`: 상단 인사이트 요약(기본 접힘, 펼치면 카드별 1문장 인사이트 + 상세)
- `PromoPanel`: 프로모션 탭 KPI/리스트
- `PartnershipPanel`: 파트너십 탭 요약
- `Modal`: 상세/클러스터 모달(바깥 클릭/Esc 닫힘은 공통 UI 컴포넌트에 의존)
- `BackToTopButton`: 스크롤 임계 이후 노출, 부드러운 상단 이동
- `ChatPanel`: 우측 패널, Enter 전송 시 고정 응답

## 에셋(Assets) 설계

- 레이아웃 실루엣: `assets/silhouettes/*.svg`
- 썸네일: `assets/thumbs/*.svg`
- `LAYOUT_LIBRARY[layoutKey].img`로 실루엣 SVG를 참조합니다.

## 문구/표기 정책

- 사용자에게 노출되는 텍스트에서 “mock” 표현은 제거합니다.
- 상단 설명 문구는 다음으로 고정합니다.
  - “검색 결과를 요약해드립니다. 검색 결과의 특징을 확인하고 다음 소재 기획에 활용해보세요.”

## 실행(로컬)

- 정적 서버로 실행합니다. 예:
  - `python3 -m http.server 5173 --bind 127.0.0.1`
- 브라우저에서 `http://127.0.0.1:5173`을 엽니다.

## 변경 이력(요약)

- ASCII 아트 실루엣 → SVG 실루엣으로 전환
- 다크모드 영향으로 인한 가시성 문제 방지(명시적 스타일/라이트 톤 유지)
- “총 N건/검색 시간”이 자연스럽게 보이도록 표시 로직 보강
- 인사이트 요약 카드 fold/expand, 카드별 1문장 인사이트 추가
- KPI 카드(매출/공헌이익) 추가 및 값 우측 정렬(라벨은 좌측 유지)
- AI Chat 패널 추가(고정 응답)
