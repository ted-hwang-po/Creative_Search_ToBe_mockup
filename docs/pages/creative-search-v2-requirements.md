# Creative Search v2 화면 요구사항(PRD-lite)

대상 페이지
- `src/pages/creative-search.jsx` (프로모션 소재 검색 결과 요약)
- `src/pages/creative-search-key-message.jsx` (성분/제품 구성 관심사 결과)

공통 데이터
- `src/mock/creative-search-dataset-v2.jsx` → `window.__APP.dataset.creativeSearchV2`

## 배경/문제 정의

사용자 피드백 요약(반영 대상):
1) 성분 강조 결과에서 **성분별 소재 특징**이 필요  
2) 공통: “클러스터 보기”를 **특징 설명 + 썸네일 그리드 모달**로 업데이트  
3) `creative-search.jsx` A/B test 제안은 **근거를 더 명확히**  
4) 공통: 레이블 조합 나열 대신 **추천 중심 + 상세는 액션 시 노출**  
5) `creative-search.jsx` A/B안은 실루엣이 아니라 **동일 상품/프로모션 기준의 실제 타입 예시(여러 애셋)**  
6) 공통: “클러스터” 용어가 어려움 → **소재 유형/타입/그룹** 등 쉬운 용어로 변경  
7) 공통: **내 브랜드 vs 경쟁**을 한눈에 비교하는 화면 구성 필요

## 목표(Goals)

- 기본 화면은 “추천” 위주로 간결하게, 근거/디테일은 **사용자 액션(클릭/펼치기)**로 제공한다.
- A/B 제안은 “왜 A/B인가”와 “무엇을 만들면 되는가(타입/애셋 예시)”가 즉시 이해되게 한다.
- 자사/경쟁 비교는 동일 축(같은 스케일)로 한 화면에서 비교 가능해야 한다.

## 공통 UX 원칙(노출 정책)

- **기본 노출은 Top 추천만**(Top 2~3).  
- 레이블/칩/분포는 기본 숨김. “근거 보기” 클릭 시에만 펼침.  
- 문구는 레이블 나열 대신 **행동 제안형**으로 작성한다.  
  - 예: “쿠폰 혜택형을 7일 A/B로 검증 권장” (O)  
  - 예: “쿠폰/혜택/긴급성/텍스트↔” (X)

## 공통 컴포넌트 요구사항

### A) 소재 유형(그룹) 모달 (기존 ‘클러스터’ 대체)

- 용어: UI에서 “클러스터” 표현 금지. 기본 표기는 “소재 유형”.
- 트리거: “유형 보기” 버튼
- 모달 레이아웃(필수)
  - 상단: 유형명 + 점수(또는 “추천”) + 닫기
  - 본문 좌: **유형 특징 설명(자연어)** + “추천 근거 보기(접힘/펼침)”
  - 본문 우: **유형에 속한 소재 썸네일 그리드(6~12개)**
  - 하단: 다음 액션(선택)
    - “소재 리스트 보기”(필터 적용)
    - “이 유형으로 A/B 제안 보기”(관련 섹션으로 스크롤)
- 데이터 요구
  - `CreativeTypeGroup.description`
  - `CreativeTypeGroup.heroThumbnailUrls[]`
  - `CreativeTypeGroup.rationale` (근거 펼침에 사용)

### B) 근거(Reason) 블록(공통)

- 컴포넌트 형태: “근거 보기” 버튼 → 펼치면 아래 3요소를 순서대로 노출
  1) **추천 이유 Top 3** (bullet)
  2) **지원 데이터**(퍼센트/표본 수 등; mock라도 수치 형태로)
  3) **대표 예시**(썸네일 2~6)
- 데이터 요구
  - `evidence.summary`, `evidence.drivers[]`, `evidence.supportingStats`, `evidence.exampleAssetIds[]`

## 페이지별 요구사항

### 1) `creative-search-key-message.jsx` — 성분별 소재 특징(요구사항 #1)

#### IA 변경
- 기존 탭(요약/분석/검색결과)은 유지하되, 다음 중 하나를 추가:
  - 옵션 1(권장): 상단 탭에 **“성분별 인사이트”** 추가
  - 옵션 2: “분석” 탭 안에 **성분 인사이트 섹션** 추가

#### 성분 인사이트 UI(필수)
- 성분 선택 UI: 칩/드롭다운(Top 5~8 성분)
- 선택 성분에 대해 아래를 노출
  - 1문장 요약(`IngredientInsight.summary`)
  - 자주 쓰는 레이아웃 Top(`commonLayouts`)
  - 자주 쓰는 카피/CTA Top(`commonCopyTypes`, `commonCTAs`)
  - “이 성분에서 잘 먹히는 시각적 특징”(visualTraits)
  - Do/Don’t(최소 2/1)
  - 예시 썸네일 그리드(6~8) + 클릭 시 상세(기존 카드/드로어 재사용 가능)

#### 데이터 매핑
- 성분별 특징은 `window.__APP.dataset.creativeSearchV2.ingredientInsights`를 사용한다.

### 2) `creative-search.jsx` — A/B 제안 근거 명확화(요구사항 #3)

#### 현재 문제
- A/B 세팅이 “점수 기반 휴리스틱”으로 보이고, 사용자에게 납득 가능한 근거가 부족함.

#### 요구사항(필수)
- A/B 카드 내에 **근거 블록** 추가(기본 접힘).
  - 근거는 하드코딩 문장 대신 `ABTestSuggestion.evidence`를 그대로 렌더링.
- “추천 소재 유형 Top 2”에도 동일한 근거 패턴을 제공.

### 3) `creative-search.jsx` — A/B안: 실루엣 대신 실제 타입 예시(요구사항 #5)

#### 요구사항(필수)
- A/B 그룹별로 “타입 2개”를 보여준다(예: UGC 리뷰형 + 혜택 그리드형).
- 각 타입은 “애셋 예시(썸네일) 2~3개”를 포함한다.
- 각 타입은 “필수 요소(required elements)”와 “변형(variants)”을 함께 제공해 **소재 기획 레벨**로 이어지게 한다.

#### 데이터 요구
- `ABTestSuggestion.groups[].creativePlan[]` 및 그 안의 `assetExamples[]`

### 4) 공통 — 레이블 나열 최소화(요구사항 #4)

#### 기본 화면
- 칩/태그/레이블을 과다 노출하지 않는다.
- “추천”과 “다음 행동”을 먼저 보여준다.

#### 상세 화면
- 레이블(칩) 노출은 근거 펼침 영역에서만.

### 5) 공통 — 용어 변경(요구사항 #6)

- UI 텍스트에서 “클러스터” 금지.
- 기본 용어: “소재 유형”
- 보조 용어(상황별): “타입”(A/B의 창작 단위), “그룹”(내부 분류 단위)

### 6) 공통 — 자사 vs 경쟁 비교 화면(요구사항 #7)

#### IA 제안(필수 1개)
- `creative-search.jsx`의 “분석” 탭에 **“자사 vs 경쟁 비교”** 섹션 추가(권장)

#### 구성(필수)
- 축 선택 토글: `compareAxes` 중 1개 선택
  - 예: 영상 비중 / 파트너십 비중 / Top 레이아웃 / Top CTA / 텍스트 버킷 / Top 유형
- 동일 스케일의 비교 UI(권장)
  - 퍼센트 축: BarRow/StackedBar 스타일로 브랜드별 동일 스케일 비교
  - 라벨 축: “자사 1줄 + 경쟁 3줄” 테이블
- 하단: “가장 큰 차이 Top 3” 자동 요약(간단 문장 + 근거 보기 링크)

#### 데이터 요구
- `CompetitiveCompareSlice.rowsByAxis`

## 빈 상태/예외 처리(프로토타입)

- 유형(그룹) 또는 성분 인사이트가 없으면:
  - “표시할 데이터가 없습니다.” + 다음 행동 CTA(예: “전체 결과 보기”)
- 비교 데이터가 부족하면:
  - 축 토글 비활성 또는 “표본 부족” 표시

## 성공 기준(프로토타입 관점)

- 사용자가 “왜 이 유형/AB가 추천되는지”를 **근거 펼침 한 번으로 이해**할 수 있다.
- “무엇을 만들면 되는지”가 **타입/애셋 예시로 구체화**되어 보인다.
- 자사 vs 경쟁 차이를 **축 1개 선택으로 즉시 비교**할 수 있다.
